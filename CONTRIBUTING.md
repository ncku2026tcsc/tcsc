# 貢獻指南（Contributing）

歡迎貢獻！送出 Pull Request 或修補前，請先讀完本頁的授權條款——它**直接影響本專案未來能否轉成更寬鬆的授權**。

## 授權與貢獻者條款（輕量 CLA）

本專案目前以 **GPL-3.0** 釋出。**送出貢獻（PR、patch、issue 附帶的程式碼）即代表你同意以下三點：**

1. **原創作品**：你的貢獻是你自己的原創作品，且你有權以下述條款授權它。
2. **現行授權**：你的貢獻以本專案當前授權（GPL-3.0）釋出。
3. **未來轉授權**：你授予專案維護者一份永久、全球、不可撤回的權利，可在未來以其他授權（包含更寬鬆的授權，例如 MIT/Apache-2.0）重新散布你的貢獻。

> 為什麼要第 3 點？本專案目前依賴的部分元件（包含中文 BERT 相關模型/函式庫）採用 GPL-3.0，因此整個專案目前必須採 GPL-3.0。未來若將這些元件替換為寬鬆授權版本，維護者希望能把整個專案轉成 MIT——但前提是所有程式碼的著作權人都同意。事先在此取得同意，可避免日後逐一回頭詢問每位貢獻者。若你不同意第 3 點，請在 PR 中明確註明，維護者會評估是否單獨處理或不合併。

（這是輕量版 CLA；不需另外簽署文件，送 PR 即視為同意。也歡迎在 commit 訊息加上 `Signed-off-by: 姓名 <email>`（DCO 風格）以示確認。）

## 流程

1. Fork → 開分支（`feature/xxx` 或 `fix/xxx`）。
2. 沿用現有程式風格與註解密度；重要變更請記到 `history.md`。
3. 與輸入法／剪貼簿／熱鍵相關的改動，請說明測試環境（Windows 版本、輸入法）。
4. 送 PR，說明動機與測試方式。

## 回報問題

- 閃退請附 `logs/crash.log`（或 exe 的 `_internal/logs/crash.log`）。
- 校正錯誤可附 `error_cases.jsonl` 的相關行（輸入／輸出／正解）。
